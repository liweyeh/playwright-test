# Playwright Test Sample

This is PoC for implementing automated testing for frontend project using Playwright and Github Actions

## Start-up Guide

1. Install dependencies:

   ```sh
   pnpm install
   ```

2. Create a .env file in the project root with the required credentials:

   ```sh
   TEST_USER_ID=login_account_for_ppe_CDA
   TEST_USER_PASSWORD=login_password
   ```

3. Run the full test suite:

   ```sh
   pnpm pw-test
   
   ```

4. Open the HTML report after a run:

   ```sh
   pnpm pw-test:report
   ```

## File Structure

.
├─ .env
├─ package.json
├─ pnpm-lock.yaml
├─ playwright.config.ts
├─ playwright-report/
├─ test-results/
└─ tests/
   ├─ assetion.spec.ts
   ├─ snapshot.spec.ts
   └─ snapshot.spec.ts-snapshots/

## What Each File/Folder Does

- .env
  - Holds environment variables for test credentials used by the tests.
- playwright.config.ts
  - Playwright test configuration (projects, reporters, retries, etc.).
- tests/
  - Test specs for Playwright.
  - assetion.spec.ts: an assertion-based login test using ARIA snapshots.
  - snapshot.spec.ts:
    a screenshot snapshot test (intentionally set up to fail by inputting text).
  - snapshot.spec.ts-snapshots/: baseline images for snapshot comparisons.
- playwright-report/
  - Generated HTML report output after test runs.
- test-results/
  - Raw artifacts from test runs (traces, screenshots, snapshots).

## pnpm Commands

- pnpm pw-test
  - Runs the full Playwright test suite across all configured browsers.
- pnpm pw-test:report
  - Opens the HTML report generated by the last run.
- pnpm pw-test:ui
  - Launches Playwright Test UI for interactive test runs.
- pnpm pw-test:update
  - Updates snapshot baselines after intentional UI changes.
- pnpm pw-test:tag {tags}
  - Runs tests matching a tag using Playwright grep.
  - Example: pnpm pw-test:tag @snapshot
